# 高频电商价格指数系统——架构设计与实现报告

## 一、项目概述

本项目旨在构建一套基于电商平台每日商品价格与销量数据的高频价格指数计算系统。系统处理每日商品明细数据，通过列式数据库ClickHouse实现秒级聚合计算，输出日度价格指数趋势，并提供可视化图表与数据导出功能，为宏观经济监测提供量化依据。

核心业务目标：

- 每日从阿里云OSS读取电商商品报价数据
- 计算各层级（SKU、类目、大盘）的价格指数（拉氏、派氏、费雪三种）
- 支持日度指数趋势图的可视化输出
- 支持指数数据导出（CSV/Excel）

## 二、架构设计

### 2.1 轻量化架构选型

基于"海量数据、高聚合需求、小团队无专职运维"的约束，系统采用轻量化架构设计：

| 组件     | 选型                    | 职责                          |
| ------ | --------------------- | --------------------------- |
| 对象存储   | 阿里云 OSS               | 海量原始数据存储与归档                |
| 分析型数据库 | ClickHouse            | 列式存储 + 秒级聚合查询              |
| 计算与调度  | Python                | ETL数据清洗、指数计算、流程编排          |
| 可视化    | Matplotlib            | 日度指数趋势图绘制与输出               |
| 配置管理   | python-dotenv         | 敏感配置环境变量管理                 |

**设计原则**：用对象存储承载海量原始数据（低成本、免运维），用ClickHouse内置计算能力完成聚合（避免维护计算集群），用Python脚本串联整个流程（低耦合、易维护）。

## 三、核心数据流水线

系统遵循**直接从OSS读取 → 清洗 → 导入ClickHouse → 引擎内计算 → 可视化/导出**的流水线设计：

```

                                    │

┌─────────────────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┘
│                   3. 加载到OLAP引擎 (Load to ClickHouse)               │
```

## 五、Schema 设计（ClickHouse）

### 5.1 分类维度表（category）

```sql
CREATE TABLE IF NOT EXISTS category (
    category_id   UInt64,             -- 分类ID
    category      String,             -- 分类名称
    hierarchy     UInt8,              -- 层级（1=一级, 2=二级）
    weight        Float32,            -- 权重
    price         Nullable(Float32),  -- 参考价格（一级分类可为NULL）
    parent        Nullable(UInt64),   -- 父分类ID（一级分类为NULL）
    load_time     DateTime DEFAULT now()
) ENGINE = MergeTree()
ORDER BY (hierarchy, category_id)
SETTINGS index_granularity = 8192;
```

### 5.2 商品全量表（product_full）

```sql
CREATE TABLE IF NOT EXISTS product_full (
    product_id    UInt64,             -- 商品唯一标识
    category_id   UInt64,             -- 所属分类ID
    name          String,             -- 商品名称
    weight        Float32,            -- 权重
    price         Nullable(Float32),  -- 当前价格
    change_count  UInt32,             -- 变更次数
    dt            Date,               -- 快照日期（分区字段）
    load_time     DateTime DEFAULT now()
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(dt)
ORDER BY (dt, category_id, product_id)
SETTINGS index_granularity = 8192;
```

**设计理由**：
- **分区键 `toYYYYMM(dt)`**：按月切割数据，查询时仅扫描目标月份
- **排序键 `(dt, category_id, product_id)`**：查询模式以日期+分类筛选为主，此顺序最大化索引命中率
- **ENGINE = MergeTree()**：ClickHouse最通用的表引擎，支持主键索引和数据分区

### 5.3 每日价格快照表（product_price）

```sql
CREATE TABLE IF NOT EXISTS product_price (
    product_id    UInt64,             -- 商品唯一标识
    category_id   UInt64,             -- 所属分类ID
    name          String,             -- 商品名称
    price         Nullable(Float32),  -- 当日价格
    change_date   Date,               -- 价格日期
    load_time     DateTime DEFAULT now()
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(change_date)
ORDER BY (change_date, category_id, product_id)
SETTINGS index_granularity = 8192;
```

### 5.4 指数结果表（index_result）

```sql
CREATE TABLE IF NOT EXISTS index_result (
    date          Date,               -- 计算日期
    index_level   String,             -- 'total' / 'category'
    category_id   UInt64,             -- 分类ID（总指数时为 0）
    category_name String,             -- 分类名称（总指数时为 'ALL'）
    index_type    String,             -- 'laspeyres' / 'paasche' / 'fisher'
    index_value   Float64,            -- 指数值
    base_date     Date,               -- 基期日期
    compute_time  DateTime DEFAULT now()
) ENGINE = MergeTree()
PARTITION BY toYYYYMM(date)
ORDER BY (date, index_level, category_id);
```

## 六、核心算法实现

### 6.1 底层SKU处理：几何平均（Jevons Index）

同一SKU一天内可能存在多个报价，采用几何平均抹平极端异常值：
```sql
EXP(AVG(LN(price))) AS price_geom_avg
```

### 6.2 聚合层算法

系统计算三种经典价格指数：

1. **拉氏指数（Laspeyres Index）**：以基期商品权重为权重
   ```
   Laspeyres = Σ(p_t * w_0) / Σ(p_0 * w_0)
   ```

2. **派氏指数（Paasche Index）**：以当期商品权重为权重
   ```
   Paasche = Σ(p_t * w_t) / Σ(p_0 * w_t)
   （注：本系统通过 p_t * w_0 / p_0 近似模拟当期权重）
   ```

3. **费雪理想指数（Fisher Index）**：拉氏与派氏的几何平均
   ```
   Fisher = √(Laspeyres × Paasche)
   ```

系统同时计算大盘总指数和各分类指数两个层级。

## 七、CLI命令行接口

系统通过`python -m src.main`提供完整的命令行操作：

| 命令 | 功能 |
|------|------|
| `create-tables` | 在ClickHouse中创建所有数据表 |
| `import` | 运行完整数据导入（分类+商品+每日价格） |
| `verify` | 验证各表行数与计算状态 |
| `import-category` | 仅导入分类数据 |
| `import-products` | 仅导入商品全量数据 |
| `import-prices` | 导入每日价格数据 |
| `compute` | 计算所有日期的价格指数 |
| `compute-latest` | 仅计算最新日期的指数 |
| `run-pipeline` | 一键运行：完整导入 + 指数计算 |
| `plot-total` | 绘制总指数趋势图 |
| `plot-category` | 绘制指定分类指数趋势图 |
| `list-categories` | 列出所有可用分类 |
| `export` | 导出指数数据为CSV/Excel |

常用参数：
- `--dt YYYY-MM-DD`：指定日期
- `--base-date YYYY-MM-DD`：指定基期日期
- `--recompute`：清空结果重新计算
- `--no-show`：不弹出图表窗口，仅保存文件
